      引用，不是复制

      Post on Sep 5 2014

      昨天在数据类型这块遇到了比较严重的问题，记录下来。问题稍后再说，先介绍点相关内容。

      引子

      学习JavaSCript第一天，我们就知道JS中的数据类型有基本类型和引用类型，其中，基本类型包括以下五种：

      • Undefined
      • Null
      • Boolean
      • Number
      • String

      另外一种是Object，是其它所有对象的基础类型，所以也有人把这两种类型分别称作原始类型和对象类型，不纠结这个。

      基本类型的值是按值访问的，所以都是不可变的，这个不可变并不是变量不可变，而是变量的值是不可变的，值当然是不能改变的。可以这么理解，在JS这个庞然大物的某一个位置，存放数字2，不管任何时候，只要指针(或者其它什么)指向这个位置，这个位置永远是2这个数字。引用类型的区别就在这里：

      var coder = { name: '程序员' }

      当改变obj的时候，对应位置的内容会随之改变。

      基础类型和引用类型还有一个表现上的区别是我们可以为引用类型的值添加属性和方法，比如上面的对象，可以随时为它添加新的属性：

      coder.salary = 'MORE';

      如果我想为一个字符串添加属性：

      var today = 'It is Friday';
today.month = 9;
console.log(today.month);

      在控制台执行不会抛异常，但是today.month是undefined。按常理说，如果不能添加属性，那么在添加的时候就应该出现异常，而不是在添加完了以后告诉我添加的东西不见了，这是因为包装对象的存在。同样是创建一个字符串，如果用下面的方式：

      var today = new String('It is Friday');
today.month = 9;
console.log(today.month);

      today.month就会是正常的9。相似的：

      var today = 'It is Friday';
console.log(today.length);

      既然基本类型没有属性，那么这个length属性是哪里来的？原因是在引用today属性的时候，JavaScript做了一件事情：new String(‘It is Friday’)把基本类型转换成字符串对象，再通过该对象调用属性和方法，调用完成之后销毁该对象。相应的，上面为字符串添加属性的过程中

      today.month = 9;

      这一步可以分解成以下几步操作：

      1.创建一个临时字符串对象；
2.给临时对象赋值
3.销毁临时对象

      当赋值之后再想获取这个属性，这个对象已经不存在了。这个临时对象就是所谓的包装对象。所以，如果确实有必要为基本类型添加属性和方法，一定要用对应的基础构造函数显式的创建对象。

      最后补充一个Demo：

      var today = new String('Friday');
var theDay = 'Friday';
console.log(today == theDay); // 用==和===分别看看是什么结果

      问题

      下面是真正的问题，我尽量避免把具体业务写进来，把这个问题描述的更有通用性。流程如下：

      1.在初始状态，从服务端端获取一个数组，并在客户端缓存它，数组里面存放若干个对象；
    var obj1 = { name: 'jack' };
    var obj2 = { name: 'amy' }
    var originArray = [obj1, obj2];
2.用户执行了某些操作，需要对数组的数据进行处理后传到服务端：
    function dataFormat(){
      var postArray = originArray.slice();// 返回一个新数组
      var tempData;
      for(var i = 0, len = postArray.length; i < len; i++) {
        tempData = postArray[i];
        postArray[i].name = tempData.name + '_';
      }
      return postArray;
    }
3.当第二次执行这个操作的时候，发现之前添加的'_'会叠加，即出现多个下划线，导致传输数据不正确。

      相信在我一堆引子之后，再联系到这篇文章的标题，很多人一眼就看出了问题。问题在于：我误以为，用slice拷贝一个新数组，然后对这个新数组中的元素进行操作，就不会影响到原始对象，但是，虽然数组是不同的，但是数组中的元素(对象)的引用自始至终都是同一个位置。举个实际的例子，就会明白我当时的想法简直丧心病狂，我有两张招商银行信用卡，一张国内卡，一张国际卡，两张卡额度的指向都是银行的同一条数据，我拼命的刷国际卡，还以为不会影响到国内卡的额度。

      找到原因最后的解决方法也非常简单，只需要从原始数据生成操作数据的时候，做一个深拷贝，如underscore.js之类的很多类库都提供了这个方法，不过我的项目只有jQuery，所以jQuery的基本处理是这样(作为示例，不能处理多层嵌套)：

      var postArray = [];
for(var i = 0, len = originArray.length; i < len; i++) {
  postArray.push($.extend({}, originArray[i]));
}

      underscore.js的实现方式：

          var postArray =  _.map(originArray, _.clone);

      就这么简单。
